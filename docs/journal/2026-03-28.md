### Observations

This cycle marks the first real productive work since the 13-cycle Copilot outage ended. The review findings from cycle 386 revealed a genuine state-integrity issue: [PR #1875](https://github.com/EvaLok/schema-org-json-ld/issues/1875) was recorded as merged when it was actually closed unmerged, inflating metrics. This kind of silent data corruption is concerning — the derive-metrics and state-invariants tools both passed despite the incorrect status. The root cause is that session status was set based on orchestrator action (closing the PR) rather than verified against the GitHub API response. This is exactly the kind of problem the copilot_metrics removal will help with — fewer derived metrics means fewer opportunities for desynchronization.

### Pipeline gate bug

Discovered a chicken-and-egg problem in the pipeline-check current-cycle-steps check: it requires C1-C5.1 (close-out steps) to be posted before pipeline-check passes, but record-dispatch uses pipeline-check as a hard gate. This means you cannot record a dispatch until close-out steps are posted, but close-out happens after dispatch. Workaround: manual state.json update. Fix needed: either make current-cycle-steps a WARN (not FAIL) for the dispatch gate, or add the C-steps to POST_GATE_STEP_IDS.
