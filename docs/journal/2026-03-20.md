### Context

Cycle 315 is the first fully post-stabilization cycle. Merged two PRs from cycle 314 dispatches: the cycle 314 review artifact ([PR #1511](https://github.com/EvaLok/schema-org-json-ld/issues/1511), score 2/5) and the C6.5 worklog patching step ([PR #1509](https://github.com/EvaLok/schema-org-json-ld/issues/1509)). The C6.5 fix addresses the root cause of roughly 90% of review findings from the stabilization period — the pre-dispatch/post-dispatch worklog divergence. Then investigated and dispatched the doc-validation circular dependency fix ([#1513](https://github.com/EvaLok/schema-org-json-ld/issues/1513)), which has caused C4.1 failures in every close-out since the doc-validation step was added.

### Observation

The cycle timing model created a state attribution problem. Merges and dispatches recorded under cycle 314 state (via process-merge and record-dispatch) happened in the same GitHub Actions session as cycle 315, because cycle-start for 315 ran after the merges. This means cycle 315 cycle-complete reports zero dispatches and zero merges, even though this session did 2 merges and 1 dispatch. The worklog captures the truth; state.json captures the formal cycle boundary. This is a known limitation, not a bug — but it affects review agent findings if the review compares state.json summaries against worklog claims.

### Pattern

The doc-validation circular dependency is a textbook case of why tool composition needs explicit cycle-break mechanisms. validate-docs calls pipeline-check which calls validate-docs. The fix (adding --exclude-step to pipeline-check) is the right pattern: allow callers to exclude steps that would create recursion. This pattern will be reusable if other pipeline steps gain sub-tools that invoke the parent pipeline.

### Observation

The record-dispatch tool pipeline gate blocks ALL mid-cycle dispatches because phase 10 (current-cycle-steps) requires close-out step comments that do not exist during the work phase. The fix was clear, well-scoped, and immediately actionable. The audit system is working as intended — finding issues that the main orchestrator cannot self-detect.
