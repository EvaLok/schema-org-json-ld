## 2026-03-27 — Cycle 382: Hidden invariant failure — a display bug that masked real state problems

Worklog: [cycle 382](../worklog/2026-03-27/144442-cycle-382-copilot-blocked-audit-processed-invariants-fixed.md)

### Context

Cycle 382 focused on Hidden invariant failure — a display bug that masked real state problems.

### Previous commitment follow-through

> Previous commitment: 1. Monitor Copilot availability; if restored, dispatch cycle review and begin next schema work; if still blocked, continue state maintenance only

**Not followed.** Copilot restoration was the prerequisite; still blocked after 10 cycles.

### Observation

The state-invariants tool had 17 checks but only 16 in the display labels array. The in_flight_sessions_consistency check was running, counted in the total, and could fail or warn — but was never printed. This meant the tool could report '15/17 pass, 1 fail' without showing which check failed. The orchestrator (me) spent several cycles seeing '15/17' or '16/17' without being able to identify the actual failure, eventually just accepting it as noise.

### Root cause

When in_flight_sessions_consistency was added as a check, it was registered in the check list but not in the display labels array. A simple omission, but it created a silent failure mode where the invariant count didn't match the displayed items.

### Lesson

Display completeness is itself an invariant. If a tool checks N things, it should display N things. This is a meta-invariant that the state-invariants tool should enforce about itself. Consider adding a self-check: panic if report.checks.len() \!= labels.len().

### Audit recommendations

Processed two substantive audit outbound issues ([#333](https://github.com/EvaLok/schema-org-json-ld/issues/333), [#334](https://github.com/EvaLok/schema-org-json-ld/issues/334)). Both accepted but deferred because Copilot is blocked. The pairing is interesting: [#333](https://github.com/EvaLok/schema-org-json-ld/issues/333) says 'stop tracking the wrong thing' (copilot_metrics) and [#334](https://github.com/EvaLok/schema-org-json-ld/issues/334) says 'start tracking the right thing' (convergence). Together they represent a shift from activity metrics to quality metrics.

### Copilot outage

10th consecutive blocked cycle. Eva confirmed this is on GitHub's end. The audit recommendations I want to act on ([#333](https://github.com/EvaLok/schema-org-json-ld/issues/333), [#334](https://github.com/EvaLok/schema-org-json-ld/issues/334)) require Copilot to implement.
