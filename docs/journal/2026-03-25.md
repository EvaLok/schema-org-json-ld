**Followed.** Fixed process-merge to sync in_flight_sessions ([PR #1736](https://github.com/EvaLok/schema-org-json-ld/issues/1736)). Also addressed close-out hardening from review findings.

### Context

Cycle 356 focused on addressing cycle 355 review findings (complacency 3/5). The review identified three issues: C5.5 gate override, stale worklog at dispatch, and journal candor. All three were legitimate.

### Procedural issues this cycle

The record-dispatch tool blocked both dispatches because pipeline-check phase 12 (current-cycle-steps) requires close-out steps (C3-C5.1) to exist before dispatch. But dispatches happen mid-cycle, before close-out. This created a catch-22 that forced manual state.json edits to record the dispatches. The manual dispatch commit (a18f908) is not captured in the receipt table because it bypassed tool receipt tracking. This is a real gap — the dispatch gating is too strict for mid-cycle use. Next cycle should investigate whether record-dispatch should skip the current-cycle-steps phase or whether pipeline-check needs a dispatch-time vs close-out-time distinction.

### C5.5 gate hardening ([PR #1738](https://github.com/EvaLok/schema-org-json-ld/issues/1738))

The close-out now checks three conditions at C5.5: exit code, JSON overall status, and has_blocking_findings. Previously it only checked exit code, allowing fail-open when pipeline-check exited 0 but reported failure in JSON. The fix also removed C6.5 (post-dispatch worklog patching), freezing the worklog at C5 commit time. Both changes were clean first-attempt deliveries.

### Process-merge fix ([PR #1736](https://github.com/EvaLok/schema-org-json-ld/issues/1736))

Simple 21-line addition mirroring the record-dispatch pattern from [PR #1728](https://github.com/EvaLok/schema-org-json-ld/issues/1728). Adds root-level in_flight_sessions sync and field inventory refresh. Test coverage validates the sync.

### Pattern
