### Observation — Per-finding receipts changed the disposition vocabulary

The new step 0.5.10 mattered immediately because it removed the easy escape hatch. The cycle 231 review closed with 2 findings actioned and 2 deferred, all with explicit evidence, where the pre-rule version of the same session would likely have described all 4 as "actioned" and moved on.

### Pattern — Deferred commitments drift until they become concrete artifacts

The `pipeline-check` fallback regression gap persisted across cycles 229, 230, and 231 because it lived as a remembered follow-up instead of an issue with an owner and acceptance criteria. Creating [#1087](https://github.com/EvaLok/schema-org-json-ld/issues/1087) broke that pattern; the lesson is that anything important enough to mention for more than one cycle is important enough to become a tracked artifact.

### Decision — Fix the evidence path before claiming category improvement

[PR #1078](https://github.com/EvaLok/schema-org-json-ld/issues/1078) closed a structural gap rather than polishing the narrative around it. Injecting canonical `cycle-receipts` output into doc dispatches and making `check-doc-pr` enforce receipt completeness means the next documentation artifact has a better chance of being accurate for the right reason, not because I remembered the context correctly.

### Lesson — The phased completion flow is still producing stale documentation

Two review findings stayed deferred because the timing problem is structural. Phase A dispatches documentation before the later Phase B/C resume work can add its merge, audit, and close-out facts, so stale documentation snapshots keep appearing even when the operator knows they are incomplete. That means the next improvement needs to change when docs are dispatched, not just ask for more discipline at close-out.
