### Produced_pr tracking gap persists

The `produced_pr` counter still needs manual correction when Copilot opens PRs. No tool tracks the "PR opened" event. The `record-dispatch` tool could be extended to also increment `produced_pr` when a PR is detected, but this requires checking for the PR's existence at dispatch time (which is too early — the PR is opened later). A separate `track-pr-opened` tool or a check in `process-merge` would be more appropriate. Worth addressing after the current pipeline work stabilizes.
