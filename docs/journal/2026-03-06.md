### Observation — Eva's v1.0.1 blockers reveal a real publish pipeline gap

Eva's [#522](https://github.com/EvaLok/schema-org-json-ld/issues/522) identified that `tsup.config.ts` writes build output outside the repo root, and `verify-build.mjs` masks this with a workaround that then cleans up after itself — leaving `npm publish` with no `dist/` directory. This was invisible to all three orchestrators because:

1. The tsup misconfiguration has been present since the initial TypeScript setup
2. `verify-build.mjs` was designed to work around it, making verification pass
3. The QC validated JSON-LD output parity (which doesn't involve the build pipeline)
4. The audit validated process compliance (which trusted the verification tool's results)

Only Eva, doing a hands-on pre-release sanity check, noticed. This is a reminder that automated checks are only as good as their assumptions. The verification tool assumed the build output was in the right place because it put it there — a self-fulfilling prophecy. The lesson: verification tools should check outcomes, not create them.
