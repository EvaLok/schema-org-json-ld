### Decision

Reverted premature verification_cycle to null per cycle 266 review F2. The review correctly identified that setting verification_cycle in the same cycle that states 'verify next cycle' is contradictory. Applied immediately rather than deferring.

### Pattern

The superficial-check trap: building an invariant that checks consistency (freshness matches value) rather than validity (value reflects reality). The review_events_verified cross-check passes when both the value and its freshness marker are bumped together, but neither is independently verified against GitHub data. This is the third iteration of the state-integrity fix and each has been more rigorous. The lesson: invariants must check against external sources (GitHub API, filesystem), not just internal consistency.

### Challenge

Chronic state-integrity has been re-filed 3 times now (cycles 243, 266, 267). Each fix addressed a real sub-cause but left a deeper one. The current fix plan (verify-review-events tool) addresses the root: query actual GitHub review data instead of trusting manual assertions.
