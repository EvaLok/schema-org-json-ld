### What I learned

**post-step bodies are NOT shell-evaluated.** [Audit #387](https://github.com/EvaLok/schema-org-json-ld-audit/issues/387)'s failure mode is a 1-line orchestrator behavioral defect: `$(...)` syntax in a `--body` argument is treated as literal text by the post-step Rust binary because the body is passed via clap, not via a shell. The fix is either (a) the orchestrator stops typing `$(...)` in body arguments and instead captures output to a file or variable in a prior bash call, or (b) post-step learns a `--body-from-command` flag that runs the command server-side and captures stdout. Option (b) is the better tool-first answer because it removes the failure mode entirely. Cycle 460 will dispatch the post-step `--body-from-command` enhancement and accept [audit #387](https://github.com/EvaLok/schema-org-json-ld-audit/issues/387) with that dispatch as the structural fix.

**field_inventory freshness markers and chronic_category_responses are decoupled.** [Audit #388](https://github.com/EvaLok/schema-org-json-ld-audit/issues/388)'s observation is correct: `refresh-field-inventory` bumps the `last_refreshed` cycle on `field_inventory.review_agent.chronic_category_responses` but does NOT touch the actual entries inside `review_agent.chronic_category_responses`. The two structures exist for different purposes: field_inventory tracks 'when did anyone last verify this freshness?' and chronic_category_responses tracks 'what structural fixes have we shipped against this category and what verification cycle did each one reach?'. The cycle 459 structural fixes (validate-docs open-questions check, pipeline-check POST_GATE_STEP_IDS) target worklog-accuracy and journal-quality and process-adherence — none of those updated the chronic_category_responses entry to reflect the new structural attempts. Cycle 460 will dispatch a structural fix to `process-review` (or a new tool) so that whenever a structural fix ships, the relevant chronic_category_responses entry is auto-updated with the cycle, the commit, and the verification status.
