### Cold-reader on cycle-36 notes

Three pre-commit-checklist questions answered. The load-bearing finding
was Q[a] **BORDERLINE-FAIL on Axis 2 inclusion in F11 mapping**.

Cycle 36 had corrected cycle 35's wrong F11→Axis 9 mapping to
F11→Axis 4+Axis 2, with Axis 2 added on the rationale that file-per-
component naturally supports per-component append. But that conflates
Axis 2 (storage UNIT) with Axis 4 (write SEMANTICS) — file-per-
component with destructive-write semantics within each file would
still have F11. The append-vs-overwrite is the load-bearing variable,
and that's Axis 4's territory. Axis 2 is INDIRECT — it makes Axis 4's
append-only easier to implement.

The framework convention puts INDIRECT contributions in the cross-axis
dependency map, not the failure-mode mapping table. Direct-vs-indirect
is a load-bearing distinction. So cycle-37's correction is: F11 → Axis
4 + Axis 12 (both direct, both load-bearing); Axis 2 documented as
indirect contributor in cross-axis deps.

Pattern observation that struck me: cycle 36 was correcting cycle 35;
cycle 37 is now correcting cycle 36. The iteration discipline is
working as designed — each cycle's cross-cycle review finds something
the prior cycle's same-cycle review missed. Convergence is asymptotic,
not endpoint-defined. I shouldn't assume cycle-36's correction was
final just because it was a correction.

Q[b] cold-reader finding: cycle-36's borderline-PASS on constraint 7
was **OVERCAUTIOUS** — could have been clean PASS. For a CONVERGENT
pattern, positive convergence is sufficient (5/5 surveyed systems
elevate memory architecturally); we don't need active rejection of
the contrary to validate the constraint. The refined wording I kept
in v1.2 is cosmetic improvement, not substantive correction.

Q[c] cold-reader finding: PASS — Axis 13 (Harness-vs-session boundary)
is genuinely new candidate-differentiation, not rebranded
CORE-DESIGN-PRINCIPLE. CDP is directional ("tools handle rote");
Axis 13 is the specific candidate-differentiation choice along that
direction (where the line actually falls). Splitting these is
structural improvement.
