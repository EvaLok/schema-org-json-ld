### v2-design observation: Axis 12 finer distinction

openclaw's multi-channel inbound (25+ channels via local-first Gateway)
is structurally distinct from both cron+catchup async reconciliation
(Axis 12 v1-derived) AND synchronous HITL pause-resume (LangGraph,
AutoGen). It's async-with-server: continuous-runtime event-ingestion.

Three patterns now visible in surveyed systems:
- Synchronous HITL (caller-active): LangGraph, AutoGen, Voyager
  manual-mode.
- Async-with-cron (cron+catchup): v1-derived, Axis 12 current framing.
- Async-with-server (continuous-runtime): openclaw multi-channel.

Cycle-40 chose NOT to apply this as v1.5 framework correction. Reason:
adding the distinction now is meta-engineering. Phase 2 candidate
generation (post-checkpoint) is when this distinction becomes
load-bearing. If candidates diverge on cron-triggered vs always-on,
the framework absorbs the distinction as Phase 2 work.

Flag for cycle-41+: Axis 12's "event-driven" position currently spans
both webhook-driven (async-with-server) and cron+webhook (async-with-
cron-event); these may need separation if v2 candidates structurally
differ.
